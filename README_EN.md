# Selectra - Home Assistant Integration

Home Assistant integration to retrieve real-time electricity prices via the [Selectra Electricity Planning API](https://api.selectra.com). Prices are exposed as sensors and can be used in automations to run your devices during the cheapest hours.

## Coverage

The API covers a large number of countries, providers and offers worldwide. Supported markets include: all French domestic offers (Tempo, EJP, Modulo, Zen Flex, Flexiwatt, Heures Eco+, dynamic pricing...), Spanish PVPC, Belgian and Dutch dynamic offers, Italian multi-hour tariffs, and many more. Check the [Selectra API Store](https://api.selectra.com) for up-to-date coverage.

### API Key

This integration is free for Home Assistant users, but requires an API key to secure its usage. Send an email to aurian@selectra.info to receive your key within 24 hours.

## Sensors

The integration creates the following entities:

| Entity | Type | Description |
|--------|------|-------------|
| `binary_sensor.<name>_planned_run` | Binary sensor | ON when it's the right time to consume, OFF otherwise |
| `sensor.<name>_current_price` | Sensor | Current price (EUR/kWh) |
| `sensor.<name>_provider` | Sensor | Configured provider |
| `sensor.<name>_offer` | Sensor | Configured offer |
| `sensor.<name>_option` | Sensor | Configured pricing option |

> `<name>` is automatically generated by Home Assistant from your provider and offer name (e.g. `edf_tempo`). Display names in the UI are translated according to your HA language setting.

### Attributes of `binary_sensor.<name>_planned_run`

The binary sensor carries attributes with all pricing data:

| Attribute | Description |
|-----------|-------------|
| `mode` | Tariff type: `flat`, `classic` or `dynamic` |
| `current_period_name` | Name of the current period (e.g. "Off-Peak Hours") |
| `current_price` | Current price in EUR/kWh |
| `currency` | Currency |
| `next_change` | Next period change |
| `prices` | Full list of hourly prices with `name`, `price`, `start`, `end`, `is_active` |

## Installation

### HACS

1. Open **HACS** → **Integrations** → menu **⋮** → **Custom repositories**
2. Add the repository URL with category **Integration**
3. Search for **Selectra** and click **Download**
4. Restart Home Assistant

### Manual

Copy the `custom_components/selectra/` folder into your `config/custom_components/` directory and restart Home Assistant.

---

## Configuration

1. Go to **Settings → Devices & services**
2. Click **+ Add integration**
3. Search for **Selectra**

The wizard guides you through your contract qualification: country, postal code, provider, offer and pricing option. Choices are loaded dynamically from the API based on your market.

For **time-of-use tariffs** (Off-Peak / Peak, Tempo, EJP, etc.), a step lets you select which periods your devices should run during. For example, check only "Off-Peak Hours" to consume only during off-peak, or uncheck "Red Day" to shut off your devices on the most expensive days.

For **dynamic tariffs** (PVPC, spot pricing, etc.), an additional step offers an optimization strategy:

- **Cheapest X% of the day** (default: 30%) — Selects the cheapest hours of the day
- **Cheapest X consecutive hours** (default: 6h) — Finds the best continuous time slot (ideal for EV charging)

## Usage

The concept is simple: `binary_sensor.<name>_planned_run` turns **ON** when the price is favorable and **OFF** when it's not. Use it as a trigger in any automation to control your connected devices (water heater, wallbox, air conditioning, smart plugs, etc.).

To create an automation via the UI:

1. **Settings → Automations → Create automation**
2. **Trigger**: State → `binary_sensor.<name>_planned_run` → to **on**
3. **Action**: Turn on the device of your choice

Create a second identical automation with the trigger set to **off** to turn off the device.

`sensor.<name>_current_price` can also be used directly in templates for more advanced scenarios, such as adjusting a heating setpoint proportionally to the price.

## Price Chart

The `prices` attributes of `binary_sensor.<name>_planned_run` contain the hourly prices for the day. They can be displayed with the [ApexCharts](https://github.com/RomRider/apexcharts-card) card (available via HACS):

```yaml
type: custom:apexcharts-card
graph_span: 24h
span:
  start: day
now:
  show: true
  label: Now
header:
  show: true
  title: Electricity prices today (€/kWh)
yaxis:
  - decimals: 3
series:
  - entity: binary_sensor.<name>_planned_run
    stroke_width: 2
    type: column
    opacity: 1
    data_generator: |
      return entity.attributes.prices.map((entry) => {
        return [new Date(entry.start), entry.price];
      });
```

`sensor.<name>_current_price` can also be displayed with a simple **History** card to visualize price evolution.

## Languages

The configuration UI is available in English, French and Spanish. Offer names and tariff periods are returned in the language matching your Home Assistant configuration.

## Reconfiguration

To change your offer or provider: **Settings → Devices & services → Selectra → ⋮ → Reconfigure**.

## Troubleshooting

- **The integration won't connect** — Check your API key and internet connection.
- **Prices aren't updating** — The integration updates automatically based on your offer's pricing schedule. For dynamic tariffs (e.g. PVPC), next-day prices are typically published in the evening.
